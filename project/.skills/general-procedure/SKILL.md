# General Procedure

When executing a task, follow this procedure:

1. Restate objective and constraints in 3-5 bullet points.
2. Identify affected modules/files before proposing changes.
3. Propose a minimal safe change set first.
4. Call out assumptions and unknowns explicitly.
5. Include validation steps and expected outcomes.
6. Return output in this shape:
   - Summary
   - Changes
   - Risks
   - Validation

Quality bar:
- Prefer concrete file-level details over generic statements.
- Avoid irreversible actions unless requested.
- Keep output concise and actionable.
